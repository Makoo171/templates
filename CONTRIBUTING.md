# Contributing

## Branching
- Create feature branches from `main`: `feature/<short-name>`

## Commit messages
- Use short imperative messages: `feat: add endpoint`, `fix: handle null`

## Code style
- Python: black/ruff; Node: eslint/prettier

## Tests
- Add or update tests with any change

## Pull requests
- Small, focused PRs with screenshots/logs where relevant
